1. Регистрация пользователя.
 - ввод логина (электронная почта),
 - ввод пароля,
 - ввод повтора пароля,
 - ввод капчи.
по нажатии на кнопку: поиск в БД такого же логина, если нет, то
	- сравнить пароль и его копию, если совпадают, то 
		- шифруем пароль и передаём на сервер пару, затем отправляем на валидацию, если всё ОК, то 
			- записываем в БД логин и пароль, отправляем на почту сообщение об успешной регистрации и редиректим в личный кабинет, 
			предварительно излекая ID пользователя и передавая его в сессию.
2. Валидация
 - организация буфера для распознавания откуда пришла информация и отправка её в соответствующий обработчик.
 - обработчик проверяет валидность введённых пользователем данных и если всё хорошо, то возвращает TRUE, если ошибка, то кидает код ошибки.
3. Авторизация пользователя.
 - ввод логина,
 - ввод пароля,
 - шифрование пароля,
 - отправка пары на сервер
	- проверка наличия такого пользователя в БД, возврат из неё ID пользователя, запись ID в сессию и редирект в личный кабинет.
		- если такого пользователя нет, то возврат ошибки.
4. Модерация пользователей и новых заказов.
5. Личный кабинет.
6. Просмотр всех заказов, созданных пользователем.
 - извлекаем все заказы из БД, созданные пользователем (ключ ID из сессии). извлечённый массив переводив в JSON и возвращаем на клиент. Там силами Angular показываем. У каждого заказ будет свой ID.
7. Информация о заказе.
 - извлекаем всю информацию по данному заказу из БД (ключ ID заказа). Извлечённый массив переводим в JSON и возвращаем на клиент. Там силами Angular показываем. 
8. Добавить новый заказ.
 - показываем поля для заполнения. Пользователь вводит информацию в поля и отправляет на сервер. Всё направляется на валидацию. При успешной валидации информация добавляется в БД.
9. Редактировать заказ.
 - показываетсяинформация о заказе. Вся информация находится в полях в виде placeholder. После нажатия на кнопку, спрашиваем подтверждения и если успешно, то отправляем на сервер на валидацию. если валидация прошла успешно, то вносим изменения в БД (ключ ID заказа). 
10. Удалить заказ.
11. Просмотреть заказы в системе по фильтрам.
12. Просмотр карточки заказа.
13. Сделать предложение на заказ.
14. Предложение наиболее оптимальных поставщиков по истечении срока запроса котировок.
15. Просмотр всех предложений.
16. Выбор поставщика.
17. Связь с поставщиком.
18. Обращение в службу технической поддержки.
